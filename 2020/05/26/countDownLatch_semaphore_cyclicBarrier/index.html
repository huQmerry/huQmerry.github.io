JUC中的同步方法

Jerry.Q

java

发布于：May 26, 2020

对Java并发包中的同步方法CountDownLatch,Semaphore,CyclicBarrier进行说明和举例

CountDownLatch

解释

  • 先看哈一下源码注释，意思就是说CountDownLatch是一个同步辅助器，允许一个或者多个线程一直等到，直到一组其他线程操作完毕。

构造方法

  • 看它的构造方法

可以看到，构造方法只会传入一个count值，这个值就是用来计数的。线程会根据count值来判断是否执行。

await/countDown方法

  • await方法

    • 当某个线程调用await方法时该线程就会被阻塞，直到count值变成0，线程才会继续执行。

      ​ 这里还有个重载方法，表示该线程等待时间，如果超过unit这个时间，线程就不会等待。

  • countDown方法

    • 每当有线程调用countDown时，count值就会减1，当count等于0时所有调用await方法的线程就可以继续执行。如果count值已经为0然后调用了countDown，什么都不会发生。

CountDownLatch的使用场景通常是，一个线程等待多个线程或者多个线程等待一个线程。

举例子

  • 跑步比赛中，在所有运动员需要等待发令枪响才可以开始比赛，这里就是多个线程等待一个线程场景；而比赛结束的线程需要等到所有运动员到达才能结束比赛，这里就是一个线程等待多个线程场景。看哈代码实现：

Semphore

解释

  • 看哈源码注释第一句，说Semphore是计数信号量，从概念上讲Semphore是一组许可证(后面没贴出来，太长了。。)

    • 啥意思勒？ 就是说Semphore初始化会定义一定数量许可证，只有拿到许可证的线程才可以执行，没拿到许可证的线程会阻塞直到有线程释放许可证。

Semphore用来控制同一时间资源可被访问的线程数，通常做流量控制。

构造方法

  • 看它的构造方法

    Semphore有两个构造方法，第一个参数permits是许可证数量，第二个参数fair表示是否公平竞争(这里不细讲)，true表示公平竞争，false表示非公平，默认是false；

acquire/release方法

  • acquire

    • acquire方法获取许可证，如果没有可用许可证会等待直到有空闲许可证。

      可以看到还有一个tryAcquire方法，它会返回一个布尔值表示是否获取到了许可证。

  • release

    • 释放许可证

举例子

  • 定义一个只有三个许可证的Semphore，那么每次都只会有三个线程执行，只有其中一个线程释放了许可证，其他线程才可以争抢许可证然后执行。

CyclicBarrier

解释

  • 看哈源码注释，大概意思是说一组线程会互相等待直到共同到达屏障，并且可以被重新使用。其实从字面意思大概就可以猜到，cyclic(循环) barrier(障碍) 指的就是可以循环使用的屏障。

构造方法

  • CyclicBarrier有两个构造方法，第一个参数parties表示等待线程数量，指的是需要多少个线程准备好了障碍才会打开；第二个参数barrierAction指的是障碍打开后首先执行的线程。

await方法

  • CyclicBarrier主要的方法就是await，调用await方法该线程就会一直等待，直到达到了CyclicBarrier规定的线程数打开障碍，才可以执行。其中有一个重载方法表示在规定时间内还达不到执行状态，该线程就不直接执行。

举例子

  • 坐过山车(欢乐谷看不到头的队伍你懂的)，一群人在那排队但是规定每三人一批次上车。
  • 结果
    • CyclicBarrier可以循环使用，从结果就可以看出，当第一批人发车后，第二批人可以接着等待然后出发，接着第三批人。实现其实也很简单，构造方法时记录了count值，当到达屏障也就是count为0时，会给count重新赋值。
