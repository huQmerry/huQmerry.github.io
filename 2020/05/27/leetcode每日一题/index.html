和为K的子数组 (560题)

  • leetcode第1题和第974题都是类似题目，需要多练类似题目，量变到质变的过程。

  • 分析一下该题：

​ 看上图，我们假设pre[j] 为数组0到j之和，pre[i]为数组0到i之和。因为是连续子数组，那么如果pre[j] - K = pre[i]，那么一定有数组i到j之和为K。

  1. 以数组[1，2，4，5，2，-1 ] K为3为例，先new一个map，然后遍历这个数组把每个位置的前序之和（就是从下标0到该位置的数之和）当作map的key，value就是这个前序之和出现的次数。
  2. 假设遍历到了第j个位置，那么看一下map中有没有key为（pre[j] - k）的entry，如果有那么代表0到j之间一定有和为k的连续子数组。

把数字转换成字符串(面试题46)

  • 给定一个数字，我们按照如下规则把它翻译为字符串：0 翻译成 “a” ，1 翻译成 “b”，……，11 翻译成 “l”，……，25 翻译成 “z”。一个数字可能有多个翻译。请编程实现一个函数，用来计算一个数字有多少种不同的翻译方法。

  • 解析：数子翻译为字符串有两种

    • 要么每一个数字代表一个字符串
    • 要么相邻两个数字代表一个字符串 (相邻两个数字必须小于26)

    以12258为例子：设f(i)表示0到i子数字并且设f(-1) = 1

    • 1 -> b 一种
    • 12 -> bc、m 两种（最后相邻两个数小于26，f(i) = f(i-1) + f(i-2) = 2）
    • 122 -> bcc、bw、mc 三种 (最后相邻两个数小于26，f(i) = f(i-1) + f(i-2) = 3)
    • 1225 -> bccf、bwf、mc、bcz、mz 五种 (最后相邻两个数小于26,f(i) = f(i-1) + f(i-2) = 5)
    • 12258 -> bccfi、bwfi、mci、bczi、mzi 五种 (最后相邻两个数大于26,f(i) = f(i-1) = 5)

    从上面的例子可以看出规律：

    这其实就是斐波那且数列的变形，只是f(i)只有在最后相邻两个数字小于26时等于f(i-1) + f(i-2)，否则等于f(i-1)

每日温度(739)

  • 请根据每日气温列表，重新生成一个列表。对应位置的输出为：要想观测到更高的气温，至少需要等待的天数。如果气温在这之后都不会升高，请在该位置用 0 来代替。

    例如，给定一个列表 temperatures = [73, 74, 75, 71, 69, 72, 76, 73]，你的输出应该是 [1, 1, 4, 2, 1, 1, 0, 0]

  • 解析

    • 暴力法就是双重for循环，遍历到某一个数时，遍历这个数之后的数组找到第一个比这个数大的数，那么两数之差就是该温度需要等待的天数

    • 可以使用栈实现，这个栈是一个递减栈，栈记录的是temperatures数组的下标。遍历数组，如果当前值比栈顶元素对应的数小，将该数的下标入栈；如果当前值比栈顶对应的数大，循环出栈，栈顶数与当前值下表差就是栈顶数对应的结果值。
