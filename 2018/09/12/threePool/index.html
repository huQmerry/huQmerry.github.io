线程池笔记

Jerry.Q

java

发布于：Sep 12, 2018

对java线程池实现过程和原理解析

线程池的好处

  • 线程池使应用能够更加充分合理的协调利用cpu、内存、网络、I/O等系统资源。线程的创建需要开辟虚拟机栈、本地方法栈、程序计数器等线程私有空间。在线程销毁时需要回收这些系统资源。频繁得到创建和销毁线程会浪费大量的系统资源，增加并发编程的风险。

  • 线程池的作用包括：

    • 利用线程池管理并复用线程、控制最大并发数等。

    • 实现任务线程队列缓存和拒接策略。

    • 实现一些时间相关的功能，如定时执行、周期执行等。

    • 隔离线程环境。

线程池参数

线程池的参数包含7个，参看上面的构造方法

  • 第一个参数：

    • corePoolSize表示常驻核心线程数。执行完之后，只会销毁非核心线程，核心线程不会被销毁。

      这个值比较关键，设置过大会浪费资源，设置过小会导致线程频繁销毁创建。

  • 第二个参数：

    • maximumPoolSize表示线程池可以能够容纳同时执行的最大线程数。如果执行的线程数大于此值，之后的任务就会进入第五个参数。当corePoolSize和maximumPoolSize值相等，即为固定大小线程池。

  • 第三个参数：

    • keepAliveTime表示线程池中的线程空闲时间，当空闲时间达到KeepAliveTime值时，就会销毁线程，直到只剩下corePoolSize个线程为止，避免浪费内存和具柄资源。默认情况下，当线程数大于corePoolSize时，keepAliveTime才会起作用。

  • 第四个参数：

    • TimeUnit表示时间单位。keepAliveTime的时间单位通常是TimeUnit.SECONDS。

  • 第五个参数

    • workQueue表示缓存队列。当请求的线程数大于maxmumPoolSize时，线程进入阻塞队列。阻塞队列有无界的LinkedBlockingQueue阻塞队列、有有界的ArrayBlockQueue阻塞队列，根据不同的需求指定阻塞队列类型。

  • 第六个参数

    • threadFactory表示线程工厂。他用来生产一组相同任务的线程。在虚拟机分析的时候可以知道这个线程是哪个线程工厂创建的，好回溯定位问题。

  • 第七个参数

    • handler表示执行拒绝策略的对象。拒绝策略后面详细讲一哈。

线程池的执行过程。

  1. 当main线程执行了execute()，线程池会创建一个线程去执行该任务。
  2. 如果任务一直被创建，并且线程执行速度小于任务创建速度，那么线程池会一直创建线程直到线程数等于maximumPoolSize。
  3. 当线程数大于等于maximumPoolSize时，新创建的线程就会放入阻塞队列。如果使用的是无界阻塞队列，那么会一直创建线程直到资源耗尽；如果使用的是有界队列，当阻塞队列满了的话就会触发第七个参数(拒绝策略)。
  4. 当一切都执行完毕，线程池会销毁线程，只保留corePoolSize个线程数。

Executors

Executors中定义了5种线程池 (生产环境中不建议使用定义好的线程池，应该根据场景需要自己实现。)

  • Executors.newWorkStealingPool:

  • Executors.newCachedThreadPool:

    • corePoolSize为0，表示执行完毕后会回收所有线程
    • maximumPoolSize最大可以到达Integer.Max_VALUE，如果线程数达到Integer.Max_VALUE早就抛OOM异常了。
    • keepAliveTime默认为60秒，表示线程不工作线程存活时间。
    • 阻塞队列为SynchronousQueue，这是一个没有数据缓冲的阻塞队列。其实该线程池几乎不会用到阻塞队列，因为线程数不可能超过Integer.Max_VALUE。

  • Executors.newScheduleThreadPool:

    • corePoolSize可以通过构造方法指定。
    • maximumPoolSize最大可以到达Integer.Max_VALUE，与newCachedThreadPool类似。
    • keepAliveTime为0，表示不会回收工作线程。

  • Executors.newSingleThreadPool:

    • corePoolSize和maximumPoolSize都为1表示只会创建一个线程去执行，相当单线程串行执行。
    • keepAliveTime为0，表示不会回收工作线程。
    • 使用无界阻塞队列LinkedBlockingQueue，任务大于1的情况任务线程会进入阻塞队列等待被执行。

  • Executors.newFixedThreadPool：

    • corePoolSize和maximumPoolSize是一样的值，输入的线程数即是核心线程数也是最大线程数，不存在空闲线程。
    • keepAliveTime为0不会回收工作线程，其实keepAliveTime即使有值在这个场景下也不起作用。
    • 使用无界阻塞队列LinkedBlockingQueue，任务大于corePoolSize/maximumPoolSize的情况任务线程会进入阻塞队列等待被执行。

拒绝策略

关于拒绝策略ThreadPoolExecutor中提供了四个公开的内部静态类。

  • AbortPolicy（默认）：丢弃任务并且抛出RejectedExecutionException异常。
  • DiscardPolicy：丢弃任务，但是不抛异常，这不是推荐做法。
  • DiscardOldestPolicy：抛弃队列中等待最久的任务，然后把当前任务加入队列。
  • CallerRunsPolicy：使用上图的主线程去执行。
